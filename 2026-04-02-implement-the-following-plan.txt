⏺ Fork in Claude Code erstellt eine Kopie einer Session, von der aus du
  unabhängig weiterarbeiten kannst.

  Verwendung:

  claude --continue --fork-session

  Was passiert:

  - Neue Session-ID wird erstellt
  - Gesamter Verlauf wird kopiert
  - Original-Session bleibt unverändert
  - Du kannst in der Fork-Session einen anderen Weg einschlagen

  Typischer Anwendungsfall:

  Du arbeitest an einem Problem und Claude verfolgt Ansatz A. Du möchtest aber
  auch Ansatz B testen:

  1. Original-Session: Claude arbeitet an Ansatz A
  2. --fork-session: Du erstellst eine Kopie
  3. Fork-Session: Du testest Ansatz B – das Original bleibt unangetastet

  Unterschied zu --continue:
  ┌────────────┬────────────────┬────────────────────┐
  │            │   --continue   │   --fork-session   │
  ├────────────┼────────────────┼────────────────────┤
  │ Session-ID │ Gleiche        │ Neue               │
  ├────────────┼────────────────┼────────────────────┤
  │ Effekt     │ Fortsetzen     │ Verzweigen         │
  ├────────────┼────────────────┼────────────────────┤
  │ Original   │ Wird verändert │ Bleibt unverändert │
  └────────────┴────────────────┴────────────────────┘
  Wichtig: Wenn du in mehreren Terminals an derselben Session arbeiten willst,
  nutze --fork-session – sonst schreiben beide in dieselbe Datei und die
  Nachrichten werden durcheinander.

⏺ Fork in Claude Code erstellt eine Kopie einer Session, von der aus du
  unabhängig weiterarbeiten kannst.

  Verwendung:

  claude --continue --fork-session

  Was passiert:

  - Neue Session-ID wird erstellt
  - Gesamter Verlauf wird kopiert
  - Original-Session bleibt unverändert
  - Du kannst in der Fork-Session einen anderen Weg einschlagen

  Typischer Anwendungsfall:

  Du arbeitest an einem Problem und Claude verfolgt Ansatz A. Du möchtest aber
  auch Ansatz B testen:

  1. Original-Session: Claude arbeitet an Ansatz A
  2. --fork-session: Du erstellst eine Kopie
  3. Fork-Session: Du testest Ansatz B – das Original bleibt unangetastet

  Unterschied zu --continue:
  ┌────────────┬────────────────┬────────────────────┐
  │            │   --continue   │   --fork-session   │
  ├────────────┼────────────────┼────────────────────┤
  │ Session-ID │ Gleiche        │ Neue               │
  ├────────────┼────────────────┼────────────────────┤
  │ Effekt     │ Fortsetzen     │ Verzweigen         │
  ├────────────┼────────────────┼────────────────────┤
  │ Original   │ Wird verändert │ Bleibt unverändert │
  └────────────┴────────────────┴────────────────────┘
  Wichtig: Wenn du in mehreren Terminals an derselben Session arbeiten willst,
  nutze --fork-session – sonst schreiben beide in dieselbe Datei und die
  Nachrichten werden durcheinander.

Token-Systeme

Überblick

Token-Systeme bilden das Fundament eines Design Systems. Sie definieren wiederverwendbare Werte für Farben, Abstände, Schriften und mehr.

Wichtig: Bei der Token-Definition gibst du den Property-Typ als Suffix an (z.B. surface.bg). Bei der Verwendung reicht der Token-Name – das System ergänzt den Suffix automatisch basierend auf der Property.

name.bg: #hexFarb-Token für background
name.col: #hexFarb-Token für color
name.pad: numberSpacing-Token für padding
name.gap: numberSpacing-Token für gap
name.rad: numberSize-Token für radius
bg $nameVerwendung (Suffix wird ergänzt)

Einfache Tokens

Tokens ohne Suffix werden direkt verwendet – der Typ wird aus dem Wert abgeleitet.

Typed Tokens mit Property-Suffix

Der Property-Suffix (.bg, .col, .pad) gibt an, welche CSS-Property der Token beeinflusst. Bei der Verwendung wird der Suffix automatisch ergänzt.

Semantische Token-Struktur

Organisiere Tokens nach Bedeutung, nicht nach Wert. So kannst du später einfach Themes wechseln.

Komponenten-Tokens

Tokens können auch komponentenspezifisch sein – so bleibt alles konsistent.

Zusammenfassung

  • name.bg: value – Token-Definition mit Property-Suffix
  • bg $name – Verwendung (Suffix wird ergänzt)
  • name: value – Einfacher Token (Typ aus Wert)
  • Suffixe: .bg, .col, .pad, .gap, .rad, .boc, .fs

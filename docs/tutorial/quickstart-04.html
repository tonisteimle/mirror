States und Trigger

Was ist ein State?

In Mirror haben User Interfaces Zustände (States). Eine Interaktion kann zu einer Zustandsänderung führen. Du beschreibst, wie ein Element in jedem Zustand aussieht – das System kümmert sich um die Übergänge.

Komponente mit State definieren

Nehmen wir an, ein Button hat zwei Zustände: Normal und Selected. In der Komponenten-Definition beschreibst du beide Zustände:

Der selected: Block beschreibt, wie der Button im Zustand "selected" aussieht. Die Properties außerhalb (bg #333, col #888) gelten für den Normalzustand.

Instanz mit State starten

Du kannst eine Instanz direkt in einem bestimmten Zustand starten – schreib einfach den State-Namen dazu:

Btn "Ausgewählt" selected startet den Button direkt im selected Zustand. Das funktioniert mit jedem State, den du in der Komponente definiert hast.

Trigger: Zustand auslösen

Bisher haben wir States nur definiert, aber nicht gesagt, wann sie ausgelöst werden. Das geht mit einem Trigger – schreib ihn einfach hinter den State-Namen:

selected onclick: bedeutet: "Wenn geklickt wird, wechsle in den State selected." Klick auf den Button – er wechselt in den ausgewählten Zustand.

Toggle: Hin und zurück

Im vorherigen Beispiel bleibt der Button nach dem Klick für immer selected. Meistens willst du aber, dass ein weiterer Klick den State wieder aufhebt – wie ein Lichtschalter:

toggle macht den Unterschied: Jeder Klick wechselt zwischen on und nicht-on. Ohne toggle würde der State nur aktiviert, nie deaktiviert.

Hover: Eingebauter State

hover: ist ein eingebauter State – du brauchst keinen Trigger, das System weiß automatisch, wann die Maus über dem Element ist:

Eingebaute States: hover, focus, active, disabled. Für eigene States brauchst du Trigger.

Exclusive: Nur einer aktiv

Bei Radio-Buttons oder Tabs soll immer nur einer ausgewählt sein. Klickst du auf einen anderen, wird der vorherige automatisch deselektiert:

exclusive sorgt dafür, dass nur ein Element den State haben kann. Du beschreibst nur das Verhalten – das System kümmert sich um das Aufräumen.

Toggle vs. Exclusive

Mit toggle (ohne exclusive) kann jedes Element unabhängig an- oder ausgeschaltet werden – wie Checkboxen:

exclusiveNur einer kann aktiv sein (Radio)
toggleJeder kann unabhängig an/aus sein (Checkbox)

State durch anderes Element auslösen

Manchmal soll ein Element auf den State eines anderen Elements reagieren. Ein Menü erscheint, wenn ein Button aktiv ist:

visible when Button open: bedeutet: "Sei sichtbar, wenn der Button im State open ist." Das Frame reagiert automatisch auf den State des Buttons.

Praktisch: Toggle Switch

Ein klassischer Toggle-Switch kombiniert mehrere Konzepte. Der äußere Container hat den State, die inneren Elemente reagieren darauf:

Der innere Knob hat einen on: Block – er reagiert auf den State des Parent-Elements. Wenn das Parent on ist, ändert sich das Margin und der Knob rutscht nach rechts.

Interaktion

States, Trigger und Abhängigkeiten

Mirror ist deklarativ – auch für Interaktionen. Du beschreibst Zustände (States) und wann sie eintreten (Trigger). Die Syntax: statename [modifier] [trigger]:

System-States

Für hover, focus, active und disabled ist der Trigger implizit – du brauchst nur den State-Namen:

hover: ist die Kurzform für hover onhover:. Der State hover wird automatisch bei Mouse-Enter aktiviert und bei Mouse-Leave deaktiviert.

Custom States mit Trigger

Für eigene States musst du den Trigger explizit angeben. Der häufigste: onclick:

Ein Klick aktiviert den State selected. Die Properties im Block (bg #2563eb, col white) werden angewendet.

Exclusive: Radio-Verhalten

Mit exclusive kann nur ein Element in einer Gruppe den State haben – wie Radio-Buttons:

Klickst du auf "Option B", wird automatisch "Option A" deselektiert. Kein manuelles Aufräumen nötig.

Toggle: An/Aus wechseln

Mit toggle wechselt der gleiche Klick zwischen aktiviert und deaktiviert – wie ein Lichtschalter:

Ohne toggle würde der State bei Klick nur aktiviert, nie deaktiviert. toggle macht ihn zum Schalter.

Im Gegensatz zu exclusive (nur einer aktiv) kann mit toggle jedes Element unabhängig an/aus sein:

State-Abhängigkeiten (when)

Mit when reagiert ein Element auf den State eines anderen Elements. Das ist deklarativer als Events – du beschreibst die Abhängigkeit, nicht die Aktion:

visible when Button open: bedeutet: "Wechsle in State visible, wenn Button im State open ist." Der State-Block enthält die Properties für diesen Zustand (hier opacity 1). Die Text-Kinder gehören zum Frame, nicht zum State-Block.

Das ist mächtiger als es aussieht. Mehrere Elemente können auf denselben State reagieren:

Ein State, mehrere Reaktionen. Keine Event-Verkabelung nötig.

Animationen

State-Übergänge können animiert werden – mit Preset oder Duration:

selected onclick: bounce spielt die Bounce-Animation beim Aktivieren. 0.2s animiert alle Properties mit 200ms Transition.

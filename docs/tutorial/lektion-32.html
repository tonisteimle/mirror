Named Instances

Überblick

Named Instances sind vorkonfigurierte Komponenten-Instanzen mit festen Werten. Im Gegensatz zu Definitionen (: =) sind sie sofort verwendbar (=).

Name: = ...Definition (Template)
Name = ...Named Instance (vorkonfiguriert)

Definition vs. Instance

Der Unterschied zwischen Definitionen und Named Instances:

UserCard braucht bei jeder Verwendung Inhalt. AdminUser ist vorkonfiguriert.

Übung

Erstelle vorkonfigurierte Notification-Instanzen (Success, Error, Info):

Lösung 
// Notification Template
Notification: = Frame hor, gap 12, pad 12, bg #1a1a1a, rad 8, bor 0 0 0 3
  Icon "info", ic white, is 20
  Text "Nachricht", col white, fs 14, grow
  Icon "x", ic #888, is 16, cursor pointer

// Vorkonfigurierte Instanzen
SuccessNotification = Notification boc #10b981
  Icon "check-circle", ic #10b981, is 20
  Text "Erfolgreich gespeichert!", col white, fs 14, grow
  Icon "x", ic #888, is 16, cursor pointer

ErrorNotification = Notification boc #ef4444
  Icon "x-circle", ic #ef4444, is 20
  Text "Ein Fehler ist aufgetreten.", col white, fs 14, grow
  Icon "x", ic #888, is 16, cursor pointer

InfoNotification = Notification boc #2563eb
  Icon "info", ic #2563eb, is 20
  Text "Neue Updates verfügbar.", col white, fs 14, grow
  Icon "x", ic #888, is 16, cursor pointer

WarningNotification = Notification boc #f59e0b
  Icon "alert-triangle", ic #f59e0b, is 20
  Text "Speicher fast voll.", col white, fs 14, grow
  Icon "x", ic #888, is 16, cursor pointer

// Test
Frame w 300, gap 8
  SuccessNotification
  ErrorNotification
  InfoNotification
  WarningNotification

Zusammenfassung

  • Name: = ... – Definition (Template, braucht Inhalt)
  • Name = ... – Named Instance (vorkonfiguriert)
  • Instanzen sind sofort verwendbar
  • Ideal für: Buttons, Icons, Badges, Notifications
  • Reduziert Wiederholung im Code

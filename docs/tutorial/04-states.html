States und Trigger

Diese Seite zeigt die geplante State-Syntax. Interaktive Playgrounds folgen nach der Implementierung.

Hover

Der einfachste State. Das System weiß automatisch, wann die Maus über einem Element ist:

Btn: pad 12 24, rad 6, bg #333, col white, cursor pointer
  hover:
    bg #2563eb
    scale 1.02

Btn "Hover mich"

hover: definiert die Styles wenn die Maus darüber ist. Weitere System-States: focus: und active:.

Toggle

Mit toggle() wechselt jeder Klick zwischen Base und einem State:

Btn: pad 12 24, rad 6, bg #333, col white, cursor pointer
  on:
    bg #2563eb
  onclick: toggle()

Btn "An/Aus"

Klick-Reihenfolge: Base → on → Base → ... Base ist der Normalzustand – kein extra "off" State nötig.

Instanz im State starten

Der State wird wie ein Flag angegeben:

Btn "Normal"              // startet in Base
Btn "Bereits aktiv", on   // startet in "on"

Praktisch: Toggle Switch

Switch: w 44, h 24, rad 12, bg #333, cursor pointer
  Frame w 20, h 20, rad 10, bg white, margin 2
  on:
    bg #2563eb
    Frame w 20, h 20, rad 10, bg white, margin 2 2 2 22
  onclick: toggle()

Frame hor, gap 16, bg #1a1a1a, pad 16, rad 8
  Text "Dark Mode", col white, fs 14
  Switch

Der State kann andere Kinder haben als die Base – wie Figma Variants.

Cycle

Für mehr als zwei States verwendet man cycle():

StatusBtn: pad 12 24, rad 6, col white, cursor pointer
  todo:
    bg #333
    Icon "circle", ic white, is 14
  doing:
    bg #f59e0b
    Icon "clock", ic white, is 14
  done:
    bg #10b981
    Icon "check", ic white, is 14
  onclick: cycle()

StatusBtn "Task 1"

Klick-Reihenfolge: todo → doing → done → todo → ... Erster definierter State = Initial. Definitionsreihenfolge = Ausführungsreihenfolge.

Exclusive

Bei Tabs soll immer nur einer aktiv sein. exclusive() macht das automatisch:

Tab: pad 12 20, rad 6, bg #333, col #888, cursor pointer
  active:
    bg #2563eb
    col white
  onclick: exclusive()

Frame hor, gap 4, bg #1a1a1a, pad 4, rad 8
  Tab "Home"
  Tab "Projekte", active
  Tab "Settings"

Klickst du einen Tab, wird er active – alle anderen automatisch zurück zu Base.

Gruppierung: Geschwister des gleichen Komponententyps bilden automatisch eine Gruppe.

Auf andere Elemente reagieren

Ein Element kann auf den State eines anderen reagieren. Gib dem Element einen Namen:

Frame gap 12
  Button name MenuBtn, pad 10 20, rad 6, bg #333, col white
    open:
      bg #2563eb
    onclick: toggle()

  Frame bg #1a1a1a, pad 12, rad 8, gap 4, hidden
    MenuBtn.open:
      visible
    Text "Dashboard", col white, fs 14, pad 8
    Text "Einstellungen", col white, fs 14, pad 8

MenuBtn.open: bedeutet: "Wenn MenuBtn in open ist, wende diese Styles an."

Eigene Funktionen

Für komplexe Logik schreibst du eigene JavaScript-Funktionen:

Btn: pad 12 24, bg #333
  on:
    bg #2563eb
  onclick: handleClick()

function handleClick(element) {
  element.toggle()        // eingebautes Toggle
  analytics.track()       // eigene Logik
  sidebar.state = 'open'  // andere Elemente
}

Regel: Simpel → onclick: toggle(). Komplex → eigene Funktion, darin element.toggle().

Für API-Calls, Timer, Validation → siehe JavaScript Integration.

Praktisch: Accordion

Panel: bg #1a1a1a, rad 8, clip
  Frame hor, spread, pad 16, cursor pointer
    Text "Mehr anzeigen", col white, fs 14
    Icon "chevron-down", ic #888, is 18
  open:
    Frame hor, spread, pad 16, cursor pointer
      Text "Weniger anzeigen", col white, fs 14
      Icon "chevron-up", ic #888, is 18
    Frame pad 0 16 16 16, gap 8
      Text "Hier ist der versteckte Inhalt.", col #888, fs 13
  onclick: toggle()

Panel

Das Wichtigste

hover:System-State (auch: focus, active)
on:Custom State definieren
Btn "Text", onInstanz im State starten
onclick: toggle()Base ↔ State
onclick: cycle()States in Definitionsreihenfolge
onclick: exclusive()Nur einer aktiv in Gruppe
onclick: myFn()Eigene Funktion
element.toggle()In JS: eingebautes Toggle
MenuBtn.open:Auf State reagieren

Prinzipien: Base = Default-Start · Erster State = Initial · Simpel → eingebaut, Komplex → eigene Funktion

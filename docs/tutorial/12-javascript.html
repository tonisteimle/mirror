JavaScript Integration

Externe Logik für States und Daten

Diese Seite zeigt die geplante Syntax. Interaktive Playgrounds folgen nach der Implementierung.

Wann JavaScript?

Mirror bietet eingebaute Funktionen für einfache Interaktionen:

  • hover: – System-States (kein onclick nötig)
  • onclick: toggle() – Toggle
  • onclick: cycle() – Cycle
  • onclick: exclusive() – Tabs

Für alles andere schreibst du eigene JavaScript-Funktionen: API-Calls, Timer, Validation, Cross-Element-Steuerung – wie in Visual Basic oder Delphi.

Elemente benennen

Mit name gibst du einem Element einen Namen für den JavaScript-Zugriff:

Button name saveBtn, pad 12 24, rad 6, bg #2563eb, col white
  "Speichern"
  loading:
    "Wird gespeichert..."
    bg #666
  success:
    "Gespeichert!"
    bg #10b981

Im JavaScript ist saveBtn direkt verfügbar – ohne querySelector. Die States werden von JavaScript gesetzt, nicht von UI-Events.

State per JavaScript setzen

Mit element.state = 'stateName' setzt du den State von außen:

async function save() {
  saveBtn.state = 'loading'

  await fetch('/api/save', { method: 'POST' })

  saveBtn.state = 'success'

  // Nach 2 Sekunden zurück zu Base
  setTimeout(() => {
    saveBtn.state = 'base'
  }, 2000)
}

Das ist der Schlüssel: Mirror definiert wie jeder State aussieht, JavaScript bestimmt wann der State wechselt.

Funktionen aufrufen

Mit onclick: functionName() rufst du JavaScript-Funktionen auf:

Button name saveBtn, pad 12 24, bg #2563eb, col white
  "Speichern"
  loading:
    "..."
    bg #666
  onclick: save()

async function save(element) {
  element.state = 'loading'
  await fetch('/api/save')
  element.state = 'base'
}

onclick: save() ruft die Funktion save() auf. Mirror übergibt automatisch das Element als ersten Parameter.

Eingebaute + eigene Logik

Die eingebauten Funktionen sind auch Methoden auf dem Element:

Btn: pad 12 24, bg #333
  on:
    bg #2563eb
  onclick: handleClick()

function handleClick(element) {
  element.toggle()        // eingebautes Toggle
  analytics.track()       // eigene Logik
  sidebar.state = 'open'  // andere Elemente
}

Regel: Simpel → onclick: toggle(). Komplex → eigene Funktion, darin element.toggle().

element.toggle()Base ↔ State
element.cycle()Nächster State
element.state = 'x'State direkt setzen
element.stateAktuellen State lesen

Properties lesen und setzen

Benannte Elemente haben Punkt-Notation für häufige Properties:

// Text und Werte
label.text = 'Neuer Text'
input.value = 'Vorausgefüllt'

// Styles
box.bg = '#2563eb'
box.col = 'white'
box.opacity = 0.5

// Sichtbarkeit
panel.visible = true
panel.hidden = false

// State
button.state = 'loading'
button.state = 'base'  // zurück zum Normalzustand
.textText-Inhalt
.valueInput-Wert
.bg, .colFarben
.visible, .hiddenSichtbarkeit
.stateAktueller State
._elNatives DOM-Element

Typische Patterns

Toast mit Auto-Dismiss

Frame name toast, pad 12 20, rad 8, bg #10b981, hidden
  visible:
    visible
  Text name toastText, col white

function showToast(message) {
  toastText.text = message
  toast.state = 'visible'

  setTimeout(() => {
    toast.state = 'base'
  }, 3000)
}

Form Validation

Frame name emailField, hor, gap 8
  Input name emailInput, placeholder "Email"
    oninput: validateEmail()
  valid:
    Icon "check", ic #10b981
  invalid:
    Icon "x", ic #ef4444

function validateEmail(element) {
  if (emailInput.value === '') {
    emailField.state = 'base'
  } else if (emailInput.value.includes('@')) {
    emailField.state = 'valid'
  } else {
    emailField.state = 'invalid'
  }
}

Daten laden

Frame name userCard
  Text "Lädt...", col #888
  loaded:
    Text name userName, col white, fs 18
    Text name userEmail, col #888
  error:
    Text "Fehler beim Laden", col #ef4444

async function loadUser() {
  // Base-State zeigt "Lädt..."

  try {
    const res = await fetch('/api/user')
    const user = await res.json()

    userName.text = user.name
    userEmail.text = user.email
    userCard.state = 'loaded'
  } catch (e) {
    userCard.state = 'error'
  }
}

Zusammenspiel Mirror + JavaScript

Die Aufgabenteilung ist klar – wie bei Visual Basic oder Delphi:

Mirror UI und States deklarativ definieren
Events Rufen immer Funktionen auf (mit Klammern)
JavaScript Logik, State-Wechsel, API-Calls

Eingebaute Funktionen:

  • toggle() – Wechselt zwischen Base und State
  • cycle() – Durchläuft mehrere States
  • exclusive() – Nur einer in Gruppe aktiv

Eigene Funktionen für alles andere:

  • Cross-Element → menuBtn.state = 'open'
  • API-Responses → element.state = 'loaded'
  • Timer → setTimeout(() => element.state = 'base', 3000)
  • Validation → field.state = isValid ? 'valid' : 'invalid'

Alle Events sind Funktionsaufrufe – mit Klammern.

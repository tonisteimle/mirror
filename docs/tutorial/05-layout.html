Layout

Elemente anordnen mit Flex und Grid

Das Prinzip

Das Layout-System in CSS ist über Jahrzehnte gewachsen – von float über inline-block zu flexbox und grid. Selbst erfahrene Entwickler müssen regelmäßig nachschlagen, welche Kombination von Properties zum gewünschten Ergebnis führt.

Mirror reduziert Layout auf drei Kernkonzepte:

Richtung: Kinder fließen vertikal (Standard) oder horizontal (hor).

Größe: Breite und Höhe (w, h) können ein fester Wert sein (w 200), sich dem Inhalt anpassen (hug), oder den verfügbaren Platz füllen (full).

Ausrichtung: Kinder werden mit center zentriert, mit spread verteilt, oder an einer von 9 Positionen platziert.

Dazu kommen zwei Werkzeuge für spezielle Fälle: wrap für Zeilenumbruch und grid N für gleichmäßige Spalten.

Vertikales Layout (Standard)

Menschen lesen von oben nach unten. Formulare, Listen, Artikel – fast alles folgt diesem Muster. Deshalb sind Frames in Mirror standardmäßig vertikal: Kinder werden untereinander angeordnet.

gap definiert den Abstand zwischen den Kindern – konsistent und ohne individuelle Margins.

Horizontales Layout

Navigation, Toolbars, Button-Gruppen – vieles muss nebeneinander stehen. Mit hor wechselst du die Richtung:

Vertikale und horizontale Frames lassen sich beliebig verschachteln – so entstehen komplexe Layouts aus einfachen Bausteinen.

Zentrieren

Zentrieren war in CSS lange umständlich – margin: auto, transform: translate(-50%), align-items plus justify-content. In Mirror reicht ein Wort: center zentriert Kinder auf beiden Achsen gleichzeitig.

Manchmal willst du nur eine Achse zentrieren – etwa Text vertikal zentriert, aber links ausgerichtet. Dafür gibt es hor-center (nur horizontal) und ver-center (nur vertikal).

Verteilen mit spread

Header haben links ein Logo, rechts einen Button. Footers haben Copyright links, Links rechts. Dieses Muster – Elemente an die Ränder drücken – ist so häufig, dass Mirror dafür spread hat:

spread entspricht dem CSS justify-content: space-between – aber in einem Wort.

9 Positionen

Stell dir den Container als 3×3 Raster vor: oben/mitte/unten und links/mitte/rechts. Die Kombination ergibt 9 mögliche Positionen. Die Kürzel sind intuitiv: t = top, b = bottom, l = left, r = right, c = center.

tltop-left
tctop-center
trtop-right
clcenter-left
centercenter (beide Achsen)
crcenter-right
blbottom-left
bcbottom-center
brbottom-right

Wrap

Was passiert, wenn mehr Elemente da sind als in eine Zeile passen? Standardmäßig werden sie gestaucht oder laufen über. Mit wrap brechen sie stattdessen in die nächste Zeile um – perfekt für Tag-Listen, Bildergalerien oder Filter-Chips.

Ändere die Breite des Containers im Code – die Elemente passen sich automatisch an.

Grid

Manchmal reicht Flex nicht: Du willst exakt 3 Spalten, gleichmäßig verteilt, egal wie viele Elemente. Dafür gibt es grid N – es erstellt ein Raster mit N gleichbreiten Spalten.

Kinder werden automatisch auf die Spalten verteilt. Passen nicht alle in eine Zeile, beginnt die nächste. Anders als bei wrap sind die Spalten immer gleich breit.

Praktisches Beispiel: App Layout

Zum Abschluss ein komplettes App-Layout: eine fixe Sidebar links, der Hauptbereich füllt den Rest. Horizontal angeordnet (hor), die Sidebar hat eine feste Breite (w 160), der Main-Bereich füllt den verfügbaren Platz (w full).

Beachte, wie w full sowohl im Main-Bereich als auch bei den Widgets funktioniert – auf verschiedenen Verschachtelungsebenen füllt es jeweils den verfügbaren Raum.

Zusammenfassung

  • hor – Horizontal anordnen
  • gap N – Abstand zwischen Kindern
  • center, spread – Ausrichtung
  • tl bis br – 9 Positionen
  • w 200, hug, full – Größen
  • wrap – Umbruch bei Überlauf
  • grid N – N-spaltiges Grid

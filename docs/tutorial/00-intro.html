Warum Mirror?

Eine Sprache für UIs – für Designer, nicht nur für Entwickler.

Das Problem

HTML und CSS waren nie für Anwendungen gedacht. Sie wurden für Dokumente entwickelt – Text mit Links und Formatierung.

Deshalb entstanden Frameworks wie React, Vue und Angular. Sie lösen echte Probleme: Komponentisierung, State-Management, reaktive Updates.

Aber sie richten sich an Entwickler – mit komplexer Syntax, Build-Tools und Boilerplate. Ein einfacher Button braucht 15+ Zeilen Code.

Die Lösung

Mirror ist eine Beschreibungssprache, die radikal einfach ist:

  • Tokens definieren Werte (Farben, Abstände, Schriften)
  • Komponenten nutzen Tokens und sind wiederverwendbar
  • Layouts enthalten Komponenten mit klarer Hierarchie

Kein Overhead, keine Magie. Was du schreibst, ist was du siehst.

Der Unterschied

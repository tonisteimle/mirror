Built-in Functions

Interaktionen ohne JavaScript-Code

Beim Prototyping brauchst du immer wieder die gleichen Interaktionen: Buttons, die ihren Zustand wechseln. Dropdowns, die sich unter einem Trigger öffnen. Dialoge, die zentriert erscheinen. Tabs, die Inhalte wechseln. Statt dafür jedes Mal JavaScript zu schreiben, bietet Mirror Built-in Functions – vorgefertigte Funktionen für typische UI-Patterns.

toggle() – Zustände wechseln

toggle() ist die am häufigsten verwendete Funktion. Sie wechselt zwischen States und passt ihr Verhalten automatisch an:

  • 1 Custom State: Binärer Wechsel zwischen Base und dem State
  • 2+ Custom States: Cycle durch alle States in Definitionsreihenfolge

exclusive() – Nur einer aktiv

Bei Tabs oder Radio-Buttons soll immer nur ein Element aktiv sein. exclusive() aktiviert das geklickte Element und deaktiviert automatisch alle Geschwister der gleichen Komponente.

show() / hide() – Elemente ein-/ausblenden

Mit show(ElementName) und hide(ElementName) steuerst du die Sichtbarkeit eines Elements direkt. Das Element braucht ein name-Attribut oder einen Komponentennamen.

navigate() – Inhalte wechseln

Mit navigate(ViewName) baust du Tab-Navigation oder Seitenansichten. Die Funktion zeigt die Ziel-Komponente und versteckt alle Geschwister-Komponenten im gleichen Container.

navigate() sucht nach einem Element mit data-component="ViewName" oder name ViewName, zeigt es an und versteckt alle Geschwister mit einem Komponentennamen.

navigateToPage() – Externe Seiten laden

Mit navigateToPage(dateiname) lädst du eine separate .mirror-Datei in einen Container. Ideal für Multi-Page-Prototypen:

NavLink: Button "Dashboard"
  onclick: navigateToPage("dashboard")

NavLink: Button "Settings"
  onclick: navigateToPage("settings.mirror")

// Content wird hier geladen
PageContent: Frame name PageContent

select() – Listen-Auswahl

select() markiert ein Element als ausgewählt und entfernt die Auswahl von allen Geschwistern. Ideal für Listen, Menüs und Auswahlfelder.

highlightNext() / highlightPrev() – Keyboard-Navigation

Für barrierefreie Listen: highlightNext() und highlightPrev() bewegen eine visuelle Markierung mit den Pfeiltasten. selectHighlighted() wählt das markierte Element aus.

Klicke in die Liste und nutze Pfeiltasten + Enter. Das Container-Element braucht focusable.

Overlay-Funktionen

Overlay-Funktionen positionieren Elemente intelligent relativ zu einem Trigger oder zentriert als Modal. Sie kümmern sich automatisch um:

  • Auto-Flip: Kein Platz unten? Element erscheint automatisch oben.
  • Click-Outside: Klick außerhalb schließt das Overlay automatisch.
  • Escape-Taste: Modals schließen mit Escape.
  • Backdrop: Optional verdunkelt der Hintergrund den Rest der Seite.

showBelow(element, offset?) – Dropdown positionieren

Zeigt ein Element direkt unterhalb des Triggers. Wenn unten kein Platz ist, wird automatisch nach oben "geflippt". Perfekt für Dropdown-Menüs, Autocomplete und Kontextmenüs.

showAbove(element, offset?) – Über dem Trigger

Wie showBelow(), aber das Overlay erscheint oberhalb des Triggers. Nützlich für Tooltips oder wenn der Trigger am unteren Bildschirmrand ist.

showModal(element, backdrop?) – Zentrierter Dialog

Zeigt ein Element als zentrierten Dialog. Der optionale Backdrop (Standard: true) verdunkelt den Hintergrund und schließt den Dialog bei Klick.

dismiss(element) – Overlay schließen

Schließt ein Overlay und räumt auf: Versteckt das Element, entfernt den Backdrop bei Modals, und deaktiviert Click-Outside-Handler.

Weitere Funktionen

showLeft(el) Positioniert Element links vom Trigger
showRight(el) Positioniert Element rechts vom Trigger
showAt(el, pos, opts) Flexibel: Position "below", "above", "left", "right" oder "center"

Scroll-Funktionen

Scroll-Funktionen ermöglichen sanftes Scrollen zu Elementen oder innerhalb von Containern – perfekt für Anker-Navigation, Karussells und "Back to Top"-Buttons.

scrollTo(element) – Zu Element scrollen

Scrollt das Element sanft in den sichtbaren Bereich. Ideal für Anker-Navigation und Inhaltsverzeichnisse.

scrollBy(container, x, y) – Karussell-Navigation

Scrollt einen Container um einen bestimmten Offset. Perfekt für horizontale Karussells und Bildergalerien.

Weitere Scroll-Funktionen

scrollToTop(el?) Scrollt zum Anfang (Element oder Seite)
scrollToBottom(el?) Scrollt zum Ende (Element oder Seite)

Value-Funktionen (geplant)

increment() / decrement() – Zähler

Für Quantity-Picker, Bewertungen oder Paginierung. Mit optionalen Min/Max-Grenzen.

$anzahl: 1

Frame hor, gap 12, center
  Button "−", w 40, h 40
    onclick: decrement($anzahl, { min: 1 })
  Text $anzahl, fs 20, weight bold, w 40, center
  Button "+", w 40, h 40
    onclick: increment($anzahl, { max: 99 })

set(token, value) – Wert direkt setzen

Setzt einen Token auf einen bestimmten Wert. Für Reset-Buttons, Presets oder Schnellauswahl.

$size: "medium"

Frame hor, gap 8
  Button "S"
    onclick: set($size, "small")
  Button "M"
    onclick: set($size, "medium")
  Button "L"
    onclick: set($size, "large")

Form-Funktionen (geplant)

focusNext() – Zum nächsten Feld springen

Springt zum nächsten Eingabefeld. Macht Formulare mit Enter schneller ausfüllbar.

Input placeholder "Vorname"
  onkeydown enter: focusNext()
Input placeholder "Nachname"
  onkeydown enter: focusNext()
Input placeholder "Email"
  onkeydown enter: submit(Form)

submit(form) / reset(form)

Sendet ein Formular ab oder setzt alle Felder zurück.

Clipboard (geplant)

copy(text) – In Zwischenablage kopieren

Kopiert Text – ideal für Share-Links, Promo-Codes oder Code-Snippets.

$shareUrl: "https://example.com/share/abc123"

CopyBtn: Button hor, gap 8, pad 10 16
  Icon "copy", is 16
  "Link kopieren"
  copied:
    Icon "check", ic #10b981, is 16
    "Kopiert!"
    bg #10b981
  onclick: copy($shareUrl)

Eigene JavaScript-Funktionen

Built-in Functions decken Standard-Patterns ab. Für API-Calls, Validierung oder komplexe Logik schreibst du eigene Funktionen:

// Mirror übergibt automatisch das auslösende Element
async function speichern(element) {
  element.state = 'loading'

  const response = await fetch('/api/save', {
    method: 'POST',
    body: JSON.stringify(formData)
  })

  element.state = response.ok ? 'success' : 'error'
}

Faustregel: Wenn eine einzige Funktion reicht → Built-in. Wenn mehrere Schritte, Bedingungen oder externe Calls → eigene Funktion.

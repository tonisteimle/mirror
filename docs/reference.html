Mirror Referenz

Vollständige Syntax-Referenz

Was ist Mirror?

Mirror ist eine DSL (Domain Specific Language) für schnelles UI-Prototyping. Das Ziel ist es, Ideen visuell zu validieren – nicht, produktionsreife Frontends zu bauen.

Der generierte React-Code kann als Ausgangspunkt für die Weiterentwicklung mit AI-Tools dienen, ist aber nicht für den direkten Produktiveinsatz gedacht.

Abschnitt 1

Syntax Grundlagen

Zeilenstruktur

ComponentName property value property value "text content"

Einrückung

Kinder werden mit 2 Spaces eingerückt:

Parent
  Child
    Grandchild

Kommentare

// Dies ist ein Kommentar
Button "Click"  // Inline-Kommentar

Strings

Doppelte Anführungszeichen für Textinhalt:

Button "Click me"
Label "Hello World"

Zahlen

Zahlen sind standardmäßig Pixel. Verwende % für Prozent:

Box w 200       // 200px
Box w 50%       // 50%
Box h 100       // 100px
Box pad 16      // 16px

Dimension Shorthand

Die ersten zwei Zahlen nach dem Komponenten-Namen werden automatisch als Breite und Höhe interpretiert:

// Shorthand
Box 300 400 pad 16           // w 300, h 400
Card 200 pad 8               // nur w 200

// Äquivalent zu
Box w 300 h 400 pad 16
Card w 200 pad 8

// Mischen möglich
Panel 300 h 400              // w implizit, h explizit

Konsistent mit Image: Bei Image wird der String als src interpretiert, gefolgt von Breite und Höhe: Image "foto.jpg" 800 600

Farben

Hex-Notation, optional mit Alpha:

Box bg #3B82F6        // RGB (Hintergrund)
Text col #3B82F680      // RGBA (50% Transparenz)
Box bg #FFF          // Kurzform

Komponenten-Namen

Button "Click"    // Großbuchstabe → Komponente
header hor        // Kleinbuchstabe → Element

Einfaches Farbsystem: col ist immer Textfarbe, bg ist immer Hintergrundfarbe. Keine Magic-Logik basierend auf Komponententyp.

// bg = Hintergrundfarbe
Button bg #3B82F6 "Click"  // Blauer Hintergrund
Card bg #1E1E1E            // Dunkler Hintergrund

// col = Textfarbe
Text col #FFFFFF "Hello"   // Weiße Schrift
Label col #888888          // Graue Schrift

// Kombiniert: bg und col zusammen
Button bg #3B82F6 col #FFFFFF "Click"
// ↑ Blauer Hintergrund, weiße Schrift

Flacher Zugriff

Greife auf verschachtelte Elemente direkt per Namen zu, unabhängig von der Hierarchie:

// Definition mit Verschachtelung
Header: hor between ver-cen h 60 pad l-r 24
  Left: hor gap 16 ver-cen
    Logo: w 120 h 32
    Nav: hor gap 8
      Link: col #888
  Right: hor gap 12
    Avatar: w 36 h 36 rad 18

// Verwendung - direkt auf Logo zugreifen (egal wie tief)
Header
  Logo bg #FF0000              // Modifiziert Logo in Left
  Avatar bg #3B82F6            // Modifiziert Avatar in Right

Wichtig: Namen müssen innerhalb einer Komponente eindeutig sein. Bei Duplikaten gibt der Parser einen Fehler aus.

List Items (Neue Instanzen)

Verwende - um neue Instanzen eines Elements zu erstellen:

// Definition
Dropdown: ver w 200
  Trigger: pad 12 bg #2A2A3E
  Menu: ver
    Item: pad 8 12

// Verwendung - füge mehrere Items hinzu
Dropdown
  Trigger "Optionen"
  - Item "Alle anzeigen"
  - Item "Nur aktive"
  - Item "Archivierte"

Das - funktioniert wie bei YAML-Listen: Jedes - erstellt eine neue Instanz.

Kombination: Flacher Zugriff + List Items
Card: ver pad 16 gap 12
  Title: size 18 weight 600
  Tags: hor gap 8 wrap
    Tag: pad 4 8 rad 4 bg #3B82F6 size 12

// Verwendung
Card
  Title "Projekt Alpha"        // Flach: modifiziert Title

  - Tag "TypeScript"           // Neu: weiterer Tag

Abschnitt 6

Library-Komponenten

Vordefinierte interaktive Komponenten mit eingebautem Verhalten.

Verwendung mit "as"-Syntax

Library-Komponenten sollten mit der as-Syntax verwendet werden, um benannte Instanzen zu erstellen. Dies ermöglicht Wiederverwendbarkeit und Referenzierung.

Syntax

// Benannte Instanz erstellen
MeinDialog as Dialog:
  Content: pad 24
    "Dialog-Inhalt"

// Die Instanz kann referenziert werden
Button onclick open MeinDialog "Öffnen"

Beispiele

// Dialog mit benutzerdefiniertem Namen
SettingsDialog as Dialog:
  Trigger
    Button "Einstellungen"
  Content: w 400 pad 24
    "Einstellungen hier..."

// Tabs mit benutzerdefiniertem Namen
MainTabs as Tabs:
  TabList:
    Tab: "Home"
    Tab: "Profile"
  TabContent:
    Panel: "Home content"
    Panel: "Profile content"

// Select mit benutzerdefiniertem Namen
CountrySelect as Select:
  Trigger: "Land wählen"
  Content:
    Item value "de" "Deutschland"
    Item value "at" "Österreich"
    Item value "ch" "Schweiz"
Hinweis: Die direkte Verwendung von Library-Komponenten (z.B. Dialog ohne as) funktioniert, erzeugt aber eine Warnung. Die as-Syntax wird empfohlen für bessere Lesbarkeit und Referenzierbarkeit.

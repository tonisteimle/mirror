Tutorial

Mirror lernen mit einem durchgängigen Beispiel

In diesem Tutorial bauen wir eine Task-App Schritt für Schritt auf. Jeder Schritt führt ein neues Konzept ein. Am Ende hast du eine funktionierende App mit Liste, Status-Wechsel und Interaktivität.

Schritt 1

Die erste Komponente

Wir wollen einen einfachen Task anzeigen – nur Text in einer Box. 

Task "Einkaufen gehen"

Das ist alles. Eine Komponente mit Inhalt.

Die Regel

Eine Zeile = eine Komponente. Der Text in Anführungszeichen ist der Inhalt.

  • Task – Name der Komponente
  • "Einkaufen gehen" – Textinhalt
Schritt 2

Styling hinzufügen

Der Task soll wie eine Karte aussehen – mit Hintergrund, Abstand und abgerundeten Ecken. 

Task pad 16 bg #1E1E2E rad 8 "Einkaufen gehen"

Die Regel

Properties stehen zwischen Komponente und Inhalt. Jede Property hat einen Wert.

  • pad 16 – 16px Innenabstand
  • bg #1E1E2E – Hintergrundfarbe
  • rad 8 – 8px Eckenradius

Reihenfolge: Komponente → Properties → Inhalt

Schritt 3

Wiederverwendbar machen

Wir wollen mehrere Tasks mit dem gleichen Styling. Dafür definieren wir Task als wiederverwendbare Komponente.

Definition
Task: pad 16 bg #1E1E2E rad 8
Verwendung
Task "Einkaufen gehen"
Task "E-Mails beantworten"
Task "Meeting vorbereiten"

Die Regel

Ein Doppelpunkt nach dem Namen macht aus einer Verwendung eine Definition.

  • Task: – Definition (mit Doppelpunkt)
  • Task – Verwendung (ohne Doppelpunkt)
Schritt 4

Eine Liste bauen

Die Tasks sollen untereinander stehen, mit Abstand dazwischen. 

TaskList: ver gap 8
  Task "Einkaufen gehen"
  Task "E-Mails beantworten"
  Task "Meeting vorbereiten"

Die Regel

Einrückung (2 Spaces) macht Elemente zu Kindern. Layout-Properties steuern die Anordnung.

  • ver – Vertikal anordnen (untereinander)
  • gap 8 – 8px Abstand zwischen Kindern
  • Einrückung – Task ist Kind von TaskList
Schritt 5

Design-Tokens einführen

Farben und Abstände sollen zentral definiert sein, damit wir sie leicht ändern können.

Tokens definieren
$bg-card: #1E1E2E
$radius: 8
$spacing: 16
Tokens verwenden
Task: pad $spacing bg $bg-card rad $radius

Die Regel

Alles mit $ ist eine Referenz. Tokens sind globale Design-Variablen.

  • $name: wert – Token definieren
  • $name – Token verwenden
Schritt 6

Checkbox hinzufügen

Jeder Task braucht eine Checkbox links vom Text. 

Task: hor gap 12 ver-cen pad $spacing bg $bg-card rad $radius
  Checkbox: w 20 h 20 rad 4 bor 2 boc #444
  Label:
Verwendung
Task
  Checkbox
  Label "Einkaufen gehen"

Die Regel

Komponenten können Kind-Slots definieren. Bei der Verwendung werden sie gefüllt.

  • hor – Horizontal anordnen (nebeneinander)
  • ver-cen – Vertikal zentrieren
  • Checkbox: und Label: – Slots für Kinder
Schritt 7

States für die Checkbox

Die Checkbox soll zwei Zustände haben: unchecked und checked. 

Checkbox: w 20 h 20 rad 4 bor 2
  state unchecked
    boc #444
    bg transparent
  state checked
    boc #3B82F6
    bg #3B82F6

Die Regel

States definieren verschiedene visuelle Zustände einer Komponente.

  • state name – State definieren
  • Properties darunter (eingerückt) – Styling für diesen State
  • Gemeinsame Properties stehen vor den States
Schritt 8

Klick-Interaktion

Ein Klick auf die Checkbox soll den State wechseln. 

Checkbox: w 20 h 20 rad 4 bor 2 onclick toggle
  state unchecked
    boc #444
    bg transparent
  state checked
    boc #3B82F6
    bg #3B82F6

Die Regel

Events lösen Aktionen aus. toggle wechselt zwischen den States.

  • onclick – Bei Klick
  • toggle – Zwischen States wechseln
Schritt 9

Text durchstreichen wenn erledigt

Wenn die Checkbox checked ist, soll der Text durchgestrichen und ausgegraut sein. 

Task: hor gap 12 ver-cen pad $spacing bg $bg-card rad $radius
  state pending
  state done
  Checkbox: w 20 h 20 rad 4 bor 2 onclick toggle
  Label: if done then col #666 line-through

Hmm, das funktioniert so nicht direkt. Wir brauchen eine Variable die den Status trackt.

Besser: Mit Variable

Task: hor gap 12 ver-cen pad $spacing bg $bg-card rad $radius
  Checkbox onclick assign done to not $done
  Label if $done then col #666

Die Regel

Bedingte Properties mit if ... then. Variablen mit assign ändern.

  • assign done to not $done – Wert umschalten
  • if $done then col #666 – Bedingte Property
Schritt 10

Dynamische Task-Liste

Die Tasks sollen aus einer Datenliste kommen, nicht hardcoded sein.

Daten (als Token/Variable)
$tasks: [
  { title: "Einkaufen gehen", done: false },
  { title: "E-Mails beantworten", done: true },
  { title: "Meeting vorbereiten", done: false }
]
Iteration
TaskList ver gap 8
  each $task in $tasks
    Task
      Checkbox if $task.done then bg #3B82F6
      Label $task.title if $task.done then col #666

Die Regel

each iteriert über eine Liste. $item.property greift auf Felder zu.

  • each $task in $tasks – Für jeden Task
  • $task.title – Titel des aktuellen Tasks
  • $task.done – Done-Status des aktuellen Tasks
Schritt 11

Header mit Titel

Die App braucht einen Header mit Titel und Zähler für offene Tasks. 

Header: hor between ver-cen pad $spacing
  Title: size 24 weight 600
  Counter: size 14 col #888
Verwendung
App ver gap 24
  Header
    Title "Meine Tasks"
    Counter "3 offen"
  TaskList
    each $task in $tasks
      Task ...

Die Regel

between verteilt Kinder mit maximalem Abstand (Space-Between).

Schritt 12

Add-Button mit Dialog

Ein Button soll einen Dialog öffnen, um neue Tasks hinzuzufügen. 

Header
  Title "Meine Tasks"
  AddButton pad 8 16 bg #3B82F6 rad 6 onclick open AddDialog "+ Neu"

AddDialog:
  Overlay: full bg #00000080 cen
  Content: ver gap 16 pad 24 bg #1E1E2E rad 12 w 300
    Title: size 18 weight 600
    Input: pad 12 bg #2A2A3E rad 6 bor 1 boc #444
    Actions: hor gap 8 hor-r
      Cancel: pad 8 16 onclick close
      Submit: pad 8 16 bg #3B82F6 rad 6

Die Regel

Overlay-Aktionen steuern die Sichtbarkeit von Dialogen und Modals.

  • open AddDialog – Dialog öffnen
  • close – Aktuellen Dialog schließen
Ergebnis

Die komplette App

Alle Teile zusammen – eine funktionsfähige Task-App. 

// Tokens
$bg-card: #1E1E2E
$primary: #3B82F6
$spacing: 16
$radius: 8

// Komponenten
Task: hor gap 12 ver-cen pad $spacing bg $bg-card rad $radius
  Checkbox: w 20 h 20 rad 4 bor 2 boc #444
  Label:

Header: hor between ver-cen
  Title: size 24 weight 600
  AddButton: pad 8 16 bg $primary rad 6

// Layout
App ver gap 24 pad 24
  Header
    Title "Meine Tasks"
    AddButton onclick open AddDialog "+ Neu"
  TaskList ver gap 8
    each $task in $tasks
      Task
        Checkbox if $task.done then bg $primary boc $primary
        Label $task.title if $task.done then col #666

Gelernt: Komponenten, Properties, Layout, Tokens, States, Events, Aktionen, Bedingungen, Iteratoren.

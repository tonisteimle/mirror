Mirror

Was ist Mirror?

Mirror ist eine neue Sprache zur Beschreibung von User Interfaces – samt Mirror Studio, einer Entwicklungsumgebung für Designer.

HTML und CSS waren nie für Anwendungen gedacht. Deshalb entstanden React, Vue, Angular und andere Frameworks. Aber sie alle richten sich an Entwickler. JSX, Hooks, Build-Pipelines – das ist komplex. Gerade jetzt, wo AI Code immer besser versteht und schreibt, wäre es wertvoll, wenn auch Designer mit Code arbeiten könnten.

Mirror macht das möglich. Die Sprache ist so einfach, dass sie jeder lesen kann. Sie beschreibt, wie eine Oberfläche aussieht und sich verhält. Und sie kompiliert zu React, DOM oder anderen Frameworks – die Ausgabe ist flexibel.

Ein Vergleich

Das einfachste mögliche React-Setup vs. Mirror – derselbe Login-Dialog:

React (minimales Setup)

<!DOCTYPE html>
<html>
<head>
  <script src="unpkg.com/react"></script>
  <script src="unpkg.com/react-dom"></script>
  <script src="unpkg.com/babel"></script>
  <style>
    .card {
      padding: 24px;
      background: #1a1a23;
      border-radius: 12px;
      display: flex;
      flex-direction: column;
      gap: 12px;
    }
    /* + mehr CSS... */
  </style>
</head>
<body>
  <div id="root"></div>
  <script type="text/babel">
    function App() {
      return (
        <div className="card">
          <h2>Anmelden</h2>
          <input placeholder="E-Mail"/>
          <input placeholder="Passwort"/>
          <button>Login</button>
        </div>
      );
    }
    ReactDOM.createRoot(
      document.getElementById('root')
    ).render(<App/>);
  </script>
</body>
</html>

Mirror

Card
    Title "Anmelden"
    Input placeholder "E-Mail"
    Input placeholder "Passwort"
    Button "Login"

Links: 35 Zeilen, 3 externe Libraries, CSS von Hand. Rechts: 5 Zeilen, fertig.

Struktur

In Mirror beschreibt die Einrückung die Hierarchie. Das ist die einzige Regel.

LoginDialog
    Title "Anmelden"
    Form
        EmailField
        PasswordField
        SubmitButton

Was eingerückt ist, gehört zum Element darüber. Form ist ein Kind von LoginDialog. Die drei Felder sind Kinder von Form.

Der Code enthält keine unnötigen Zeichen. Keine Tags, keine Klammern, keine Semikolons. Er liest sich fast wie eine Beschreibung: "Ein Login-Dialog mit einem Titel und einem Formular, das E-Mail, Passwort und einen Button enthält."

Die Komponenten hier – LoginDialog, Form, EmailField – sind natürlich definiert. Wie das geht, kommt später.

Properties

Properties ändern das Aussehen von Komponenten. Sie stehen nach dem Namen:

Card
    padding 24
    background #1a1a23
    radius 12
    Title "Willkommen"

Das gleiche geht auch kompakt – mit Kommas getrennt und Kurzformen wie bg, pad, rad:

Card pad 24, bg #1a1a23, rad 12
    Title "Willkommen"

Die wichtigsten Properties

  • Größe – width, height (oder w, h)
  • Abstand – padding, margin, gap
  • Farbe – background, color
  • Rahmen – border, radius
  • Text – font-size, weight
  • Layout – horizontal, vertical, center

Properties sollten normalerweise keine direkten Werte wie #2271c1 enthalten. Besser sind Tokens – dazu gleich mehr.

Tokens

Tokens sind benannte Werte. Einmal definiert, überall verwendbar.

Statt Farben und Zahlen direkt in Properties zu schreiben, definiert man sie als Tokens:

// Token-Definitionen
$primary: #2271c1
$surface: #1a1a23
$sm: 8
$md: 16
$lg: 24

// Verwendung
Card
    padding $lg
    background $surface
    radius $sm

    Button
        background $primary
        padding $sm $lg
        "Speichern"

Der Vorteil: Ändert man $primary: #22c55e, werden alle Buttons grün. Keine Suchen-und-Ersetzen-Aktion nötig.

Komponenten

Komponenten sind wiederverwendbare Bausteine. Einmal definiert, beliebig oft einsetzbar.

Eine Komponente definiert man mit einem Doppelpunkt nach dem Namen:

// Definition
Card as frame:
    padding 24
    background #1a1a23
    radius 12

// Verwendung
Card
    Text "Inhalt"

Der Unterschied:

  • Card as frame: – definiert die Komponente, rendert sie aber nicht
  • Card – verwendet die Komponente

Das as frame gibt an, welches HTML-Element dahinter steht. frame wird zu <div>, text zu <span>, button zu <button>.

Vererbung

Eine Komponente kann von einer anderen erben:

// Basis
Button as button:
    padding 12 24
    radius 8

// Varianten
PrimaryButton extends Button:
    background #2271c1
    color white

DangerButton extends Button:
    background #dc2626
    color white

Mit extends übernimmt eine Komponente alle Properties der Basis. Man definiert nur, was anders ist.

Eingebaute Komponenten

Mirror bringt eine Bibliothek fertiger Komponenten mit – Buttons, Inputs, Dropdowns und mehr. Man kann sie direkt verwenden oder als Basis für eigene nehmen:

// Direkt verwenden
Select
    placeholder "Land wählen..."
    Option "Deutschland"
    Option "Österreich"
    Option "Schweiz"

// Als Basis für eigene Variante
CountrySelect as Select:
    width 280

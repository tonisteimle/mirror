Grundlagen

Syntax-Übersicht

Mirror DSL ist eine deklarative Sprache für UI-Prototyping. Jede Zeile beschreibt eine Komponente mit Properties.

ComponentName property value property value "Text"

Wichtig: Text-Inhalte müssen IMMER am Ende der Zeile stehen, in Anführungszeichen.

Einrückung = Hierarchie

2 Leerzeichen = Kind-Element

Parent
  Child
    GrandChild

Kommentare

// Kommentar auf eigener Zeile
Button "Click" // Inline-Kommentar

Benannte Instanzen

Drei Syntaxen für benannte Instanzen:

// 1. named Keyword
Button named SaveBtn "Save"
Panel named Dashboard "Dashboard"

// 2. Primitives (Input, Image, etc.)
Input Email: "Email eingeben" type email
Image Avatar: 48 48 radius 24 fit cover

// 3. as Syntax
SettingsDialog as Dialog:
  "Dialog Inhalt"

Wichtig: Nur benannte Instanzen können in Events referenziert werden!

Tokens (Design-Variablen)

Token-Definition

Tokens sind einfache Werte (Farben, Zahlen, Strings). KEINE Arrays oder Objekte!

$primary: #3B82F6
$spacing: 16
$font-main: "Inter"

Button background $primary padding $spacing font $font-main

Daten & Iteration

Data Tab Syntax

Daten werden im Data Tab definiert, NICHT in der DSL-Datei:

// Schema-Definition
Task:
  title: text
  done: boolean
  category: Category

Category:
  name: text

// Instanzen
- Category "Arbeit"
- Category "Privat"

- Task "Einkaufen", false, Category[0]
- Task "Sport", true, Category[1]

Wichtig: Die Syntax $tasks: [{...}] (JSON-Arrays) ist NICHT unterstützt! Daten müssen über den Data Tab mit Schema-Syntax definiert werden.

Iteration (each)

Iteriert über Collections aus dem Data Tab:

// In der DSL (tasks kommt aus Data Tab)
Box
  each $task in $tasks
    Card
      Title $task.title
      Checkbox $task.done

Verschachtelte Iteration:

each $category in $categories
  Section
    Title $category.name
    each $item in $category.items
      Item $item.name

Alle Properties

Jede Property mit mindestens einem Beispiel.

Text der angezeigt wird, wenn nichts ausgewählt ist.

placeholder Property

Alle Slots

Jeder Teil des Select kann individuell gestyled werden.

  • Select
    • Label Beschriftung über dem Select — nur bei label
    • Trigger Der klickbare Button
      • Value Zeigt Placeholder oder gewählten Wert
      • Arrow Dropdown-Pfeil ▼
    • Clear X-Button zum Leeren — nur bei clearable
    • Tags Container für ausgewählte Werte — nur bei multiple
      • Tag Einzelner Tag mit X zum Entfernen
    • Search Eingabefeld zum Filtern — nur bei searchable
    • Dropdown Das aufklappende Menü
      • Empty Anzeige wenn keine Treffer — nur bei searchable
      • Group Optionale Gruppierung von Items
        • GroupLabel Überschrift der Gruppe
      • Item Eine wählbare Option
        • Check Checkmark ✓ bei Auswahl

Komponenten definieren

Definiere wiederverwendbare Select-Komponenten.

Definition vs Instanz

Syntax-Unterschied
Name: = Select definiert eine Komponente (wird nicht gerendert).
Name: erstellt eine Instanz (wird gerendert).
Name as Parent: = für Vererbung.
Komponente definieren
# Definition (Doppelpunkt vor =) MySelect: = Select Trigger: bg #1a1a1a pad 10 14 rad 6 bor 1 #333 hover: bor 1 #444 Dropdown: bg #1a1a1a rad 8 shadow lg bor 1 #333 Item: pad 10 14 highlighted: bg #252525
Komponente instanziieren
# Instanz 1 MySelect: placeholder "Land..." Item "Deutschland" Item "Schweiz" # Instanz 2 MySelect: placeholder "Sprache..." Item "Deutsch" Item "Englisch" # Instanz 3 mit Override MySelect: multiple placeholder "Tags..." Trigger: bg #252525 # Override Item "Design" Item "Code"

Default Styling

Select kommt mit vollständigem Styling und Icons. Alles funktioniert sofort.

Konzept

Die Defaults sind im System hinterlegt, nicht als Mirror-Code den du importieren musst.

Wie es funktioniert
  1. Slot existiert nicht → System rendert Default mit Default-Styling
  2. Slot existiert ohne Styles → System rendert Default-Content, dein Styling wird gemerged
  3. Slot existiert mit Content → Dein Content ersetzt Default komplett
Beispiel: Arrow Slot Konzept
# Fall 1: Kein Arrow definiert Select: Item "A" # → System rendert: Arrow mit chevron-down, col #666, rotation bei open # Fall 2: Arrow mit nur Styling Select: Arrow col #fff # Nur Farbe ändern Item "A" # → System rendert: chevron-down Icon (Default), aber col #fff (dein Override) # Fall 3: Arrow mit eigenem Content Select: Arrow: Icon "chevrons-up-down" # Eigenes Icon Item "A" # → Dein Icon ersetzt das Default-Icon komplett
Beispiel: Check bei multiple Konzept
# Single-Select: Check ist Checkmark ✓ Select: Item "A" # → Check zeigt: Icon "check", erscheint bei selected # Multiple: Check wird automatisch Checkbox Select multiple: Item "A" # → Check zeigt: Checkbox mit Border, filled bei selected # Override: Eigener Check-Style Select multiple: Check rad 9: # Rund statt eckig = Radio-Style Box w 8, h 8, rad 4 Item "A"
Merge-Logik

Deine Styles werden mit den Defaults gemerged, nicht ersetzt. Wenn du nur col #fff setzt, bleiben alle anderen Default-Styles (pad, rad, etc.) erhalten.

CheckLucide checkErscheint bei selected
